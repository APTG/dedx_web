### Prompt 1: Write docs/04-feature-specs/export.md

**User** asked to write `export.md` as the authoritative export spec for the
dEdx Web redesign. The spec must consolidate export behaviour across Calculator,
Plot, and Multi-program (advanced mode), resolve two open questions deferred
from `plot.md`, and follow the Final spec conventions used across the project.

**AI response**: Read all five source specs (`README.md`, `calculator.md`,
`plot.md`, `multi-program.md`, `unit-handling.md`) plus the AI log index to
build context. Identified the two open questions in `plot.md` (SVG export,
CSV format toggle) and all deferred export detail across specs. Asked four
design questions via the AskUserQuestion button interface.

User decisions:
1. **SVG export** — single "Export image ▾" dropdown offering PNG and SVG
   (not two separate buttons, not PNG-only).
2. **CSV column layout (Plot)** — wide format only (no long/tidy toggle).
3. **Delta columns (multi-program CSV)** — raw data columns only; no delta/% export.
4. **Calculator CSV columns** — all five unified-table columns including
   "Normalized Energy (MeV/nucl)".

**AI response**: Wrote `export.md` Final v1 with:
- Common CSV conventions table (UTF-8 BOM, CRLF, 4 sig figs, no thousands sep).
- Calculator basic mode: 5-column schema, button placement, filename convention.
- Calculator advanced/multi-program: wide table, quantity-focus visibility rules,
  no delta columns, filename with program count.
- Plot image export: "Export image ▾" dropdown, PNG (2× canvas) and SVG
  (`makeSVG()`), ASCII wireframes.
- Plot CSV export: wide format, Case A (shared grid) / Case B (different grids),
  column header pattern, padding rule.
- Filename convention summary table with ASCII-safe rules.
- Accessibility contract (aria-label, role=menu, keyboard behaviour).
- Acceptance checklist (Calculator basic, Calculator advanced, Plot image,
  Plot CSV, Common).

Then updated all cross-referencing specs for consistency:

### Prompt 2: PDF export feasibility + external-data CSV design

**User** asked: (1) Can PDF export be done in a web app? (2) What are the
right design questions for external-data series with a different energy grid
in the Plot CSV?

**AI response**: Answered PDF feasibility — jsPDF for Plot (SVG embed + metadata),
jsPDF autotable or `window.print()` for Calculator. Asked 4 button questions:
Plot PDF content (multiselect), Calculator PDF method, external CSV label style,
external source metadata in CSV.

User decisions:
1. **Plot PDF content** — all of: series list, entity/unit metadata, external
   attribution, build info (commit hash + date + branch) in header; differs by
   mode (advanced adds more); user also wants user browser/OS info in advanced.
2. **Calculator PDF** — jsPDF autotable.
3. **External CSV labels** — use ext label as-is (no `[ext]` tag).
4. **External metadata in CSV** — `#` comment rows before the header row.

Then asked 4 follow-up button questions on basic/advanced PDF difference,
Calculator PDF orientation, PDF header build info, and mixed-grid rule.

User decisions:
1. **Plot PDF basic/advanced** — same layout; advanced adds more metadata:
   build info, material density, particle Z/A, program URLs for externals,
   interpolation settings, browser/OS. Both modes include date + clickable URL.
2. **Calculator PDF layout** — always portrait + page-break column split
   (STP columns page, then CSDA columns page) for wide advanced tables.
3. **PDF header build info** — same as footer badge (commit hash · date · branch);
   advanced mode only for build info; both modes include clickable page URL.
4. **Mixed grid rule** — always Case B when any ext series present.

**AI response**: Updated `export.md` to Final v2, adding:
- §5 PDF Export — Plot (button, basic content, advanced metadata block with
  wireframe, filename `dedx_plot_report.pdf`).
- §6 PDF Export — Calculator (button placement, basic content, advanced wide
  table split by quantity group, portrait layout, mode-sensitive metadata).
- §4.3 External Data in Plot CSV (always Case B, `#` comment rows, label-as-is).
- Updated §4.2 Case A condition (no ext series required).
- Updated §7 Filename Convention Summary (PDF filenames added).
- Updated §8 Accessibility (PDF buttons added).
- Extended §9 Acceptance Checklist (Calculator PDF, Plot PDF, ext CSV rules).

Then updated cross-referencing specs:
- `calculator.md` → Final v7: Export section updated, Export PDF button in

- `plot.md` → Final v4: controls bar updated (3 export controls), Export section
  updated (PDF summary + ext CSV rules), AC updated.
