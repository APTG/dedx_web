## Summary

Implemented critical missing features in EnergyInput component to meet calculator spec acceptance criteria:

### Fixes Applied

1. **Default rows (spec compliance)**
   - Changed `DEFAULT_ROWS` from `["0.1", "1.0", "10.0"]` to `["100"]`
   - Matches spec §Behavior.Default State: "Pre-filled with a single row: `100`"

2. **Enter/Tab key navigation**
   - Added `handleKeydown()` function in component
   - Enter/Tab moves focus to next row's input
   - At last row: auto-adds new row and focuses it
   - Uses `setTimeout` for focus management on new rows

3. **Always-empty-row-at-bottom**
   - `updateRowText()` auto-appends row when typing in last row
   - Matches spec: "Always-empty-row-at-bottom (auto-appends when user types in it)"

4. **Validation display**
   - Added `error?: string` field to `EnergyRow` interface
   - Parser errors now stored in state during `updateRowText()`
   - Red border styling via conditional class: `row.error ? 'border-destructive...'`
   - Error message displayed inline next to input

5. **Auto-delete empty rows**
   - Added `handleBlur()` function in state
   - Removes row on blur if empty (when multiple rows exist)
   - Single row never auto-deleted
