## Work Completed

### Slice 1: Energy Parser
- **Tests:** `src/tests/unit/energy-parser.test.ts` (26 tests)
- **Implementation:** `src/lib/utils/energy-parser.ts`
- Parses energy input strings with optional unit suffixes
- Supports all 10 valid units: eV, keV, MeV, GeV, MeV/nucl, GeV/nucl, keV/nucl, MeV/u, GeV/u, keV/u
- Returns `{value, unit}` | `{error: string}` | `{empty: true}` union type
- Case-insensitive unit matching
- Handles scientific notation (e.g., `1.5e-2`)

### Slice 2: Energy Conversions
- **Tests:** `src/tests/unit/energy-conversions.test.ts` (19 tests)
- **Implementation:** `src/lib/utils/energy-conversions.ts`
- Implements `convertEnergyToMeVperU()` and `convertEnergyFromMeVperU()`
- Uses `massNumber` (A) for MeV/nucl conversions
- Uses `atomicMass` (m_u) for MeV/u conversions
- Handles all SI prefix variants (eV, keV, MeV, GeV and their per-nucleon equivalents)
- Round-trip conversions preserve values to float precision

### Slice 3: Debounce Utility
- **Tests:** `src/tests/unit/debounce.test.ts` (8 tests)
- **Implementation:** `src/lib/utils/debounce.ts`
- Generic debounce function with cancel/flush capabilities
- Returns debounced function with `.cancel()` and `.flush()` methods
- Properly handles argument passing and return values

### Slice 4: Energy Input State
- **Tests:** `src/tests/unit/energy-input-state.test.ts` (12 tests)

- Tracks master unit and per-row mode state
- Provides `getParsedEnergies()` for validation/conversion pipeline
- Default initialization: 3 rows (0.1, 1.0, 10.0) with MeV unit

1. **Parser return type:** Union type (`ParseResult`) instead of throwing errors — allows caller to handle empty/invalid states explicitly

3. **Debounce implementation:** Custom implementation over lodash — zero dependencies, better tree-shaking
4. **State management:** Svelte 5 runes exclusively (no stores) — consistent with ADR
5. **Component testing:** @testing-library/svelte with behavior-focused queries — follows existing pattern
