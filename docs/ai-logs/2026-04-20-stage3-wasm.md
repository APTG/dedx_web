## Work Done

### 1. Created Thin C Wrappers

Created `wasm/dedx_extra.h` and `wasm/dedx_extra.c` implementing 4 wrapper functions that expose internal libdedx data without modifying the libdedx submodule:

| Function | Wraps | Purpose |
|----------|-------|---------|
| `dedx_get_ion_nucleon_number(int)` | `dedx_internal_get_nucleon()` | Mass number (A) for Z=1..112 |
| `dedx_get_ion_atom_mass(int)` | `dedx_internal_get_atom_mass()` | Atomic mass in u (e.g., 1.00794 for H) |
| `dedx_get_density(int, int*)` | `dedx_internal_read_density()` | Material density in g/cm³ |
| `dedx_target_is_gas(int)` | `dedx_internal_target_is_gas()` | Gas state predicate (1/0) |

These wrappers are required by the TypeScript wrapper contract (`docs/06-wasm-api-contract.md` §4.5).

### 2. Updated Build Script

Modified `wasm/build.sh` to:

1. Compile `wasm/dedx_extra.c` alongside `libdedx.a`
2. Add `-I/src/wasm` include path for the wrapper header
3. Export 23 additional functions required by the TypeScript wrapper:
   - Stateful API: `dedx_allocate_workspace`, `dedx_free_workspace`, `dedx_load_config`, `dedx_free_config`
   - Tools: `dedx_get_csda`, `dedx_get_inverse_stp`, `dedx_get_inverse_csda`, `convert_units`, `dedx_get_bragg_peak_stp`
   - Batch operations: `dedx_get_stp_table`, `dedx_get_csda_range_table`, `dedx_fill_program_list`, `dedx_fill_ion_list`, `dedx_fill_material_list`, `dedx_get_stp_table_size`, `dedx_get_simple_stp_for_program`
   - Metadata: `dedx_get_error_code`, `dedx_get Density`
   - Thin wrappers: `dedx_get_ion_nucleon_number`, `dedx_get_ion_atom_mass`, `dedx_target_is_gas`
4. Add `HEAPF64` to `EXPORTED_RUNTIME_METHODS` (required for F64 array reads in verification tooling)

Full export list now contains 40+ functions matching `docs/06-wasm-api-contract.md` §4.
