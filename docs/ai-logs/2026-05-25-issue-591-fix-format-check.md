1. **Hard error** — `SyntaxError` in `docs/ai-logs/2026-04-29-stage6-export-and-e2e-fixes.md`.

   which is not valid TypeScript. This aborts the entire run with a non-zero exit code.

**Fixes applied:**

### `.prettierignore` additions

| Pattern | Reason |
|---|---|
| `libdedx/` | Git submodule (C library) — formatted by its own project conventions |
| `vendor/` | All vendor git submodules — formatted by their own project conventions |
| `**/pnpm-lock.yaml` | Lock files are auto-generated by pnpm; formatting them with prettier creates noise diffs and provides no value. Excluding lock files is standard practice in all major JS ecosystems. |
| `docs/ai-logs/` | AI session logs containing raw tool output; pseudo-code truncation markers (`...` on its own line) cause a hard `SyntaxError` in prettier's TypeScript parser. These are notes, not source code. |

**pnpm-lock.yaml decision**: Exclude. Lock files are owned by the package manager, not the
project's style config. Running prettier on them produces meaningless reformats that pollute
diffs and can confuse pnpm's parser on re-read.
