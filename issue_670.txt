### 2. What the feature does

Adds a **stopping-power output unit selector** to the Advanced calculator. The STP column header becomes a dropdown offering `keV/µm`, `MeV/cm`, `MeV·cm²/g` (same set/order as the Plot page). Picking a unit re-renders every STP cell; the Range column is untouched. The choice is encoded in a shared `sunit=` URL param.

**Two design decisions (do not undo):**
1. **Unify on `sunit=`** for both calculator and plot. The plot migrated off `stp_unit=` but still **reads** `stp_unit=` as a legacy fallback.
2. **No `localStorage`.** URL `sunit=` is the only durable store; the choice also carries across in-app calculator↔plot nav via an in-memory `$state`. Consequence (intentional): a fresh tab with a bare URL shows the default `keV/µm` rather than restoring a last-used unit.

### 3. Architecture — render-time conversion

- Engine stores **raw mass STP** (`MeV·cm²/g`, as WASM returns).
- `computeRows()`/`parseRow()` in `calculator.svelte.ts` converts to the display unit via `convertStpMass(mass, density, unit)`.
- `getStpDisplayUnit()` returns the explicit user override (shared `stpOutputUnit` store) when set, else the existing aggregate-state default (gas→`MeV·cm²/g`, condensed→`keV/µm`). Pre-existing URLs render identically.
- A unit switch is a pure re-render (no WASM recompute), unifying single-entity with the multi-entity/plot paths.
